# 02-02: Lazy Loading Implementation

<objective>
Implement lazy loading for below-fold images to reduce initial page load and improve Largest Contentful Paint (LCP).
</objective>

<context>
@websites/www.dinero.cash/index.html
@websites/www.dinero.cash/users.html
@websites/www.dinero.cash/agents.html
@.planning/STATE.md
</context>

<tasks>
<task name="Audit all images for loading priority">
1. Open `index.html`
2. Identify above-fold images (visible without scrolling):
   - First hero slide image (hero-1.jpg) - CRITICAL, should load immediately
   - Logo in navigation - CRITICAL
3. Identify below-fold images (load after initial render):
   - Hero slides 2, 3, 4
   - Tab interface images
   - All feature card images
4. Create list of images by priority
5. Repeat for users.html and agents.html
</task>

<task name="Add loading="eager" to critical first image">
1. Open `index.html`
2. Find first hero image (hero-1 or hero-1.webp in picture element)
3. Add attributes to the `<img>` tag:
   ```html
   <img src="assets/images/hero/hero-1.jpg"
        loading="eager"
        fetchpriority="high"
        decoding="sync"
        alt="...">
   ```
4. Save index.html
</task>

<task name="Add loading="lazy" to below-fold images">
1. Open `index.html`
2. Find hero images 2, 3, 4
3. Add `loading="lazy"` and `decoding="async"`:
   ```html
   <img src="assets/images/hero/hero-2.jpg"
        loading="lazy"
        decoding="async"
        alt="...">
   ```
4. Find all tab interface images (tabs-users, tabs-business, tabs-banks)
5. Add `loading="lazy"` and `decoding="async"` to each
6. Find all feature card images (24 images in wallet/, payments/, merchant/, onboarding/)
7. Add `loading="lazy"` and `decoding="async"` to each
8. Save index.html
9. Repeat process for users.html and agents.html
</task>

<task name="Verify lazy loading behavior">
1. Open Chrome DevTools â†’ Network tab
2. Enable "Disable cache"
3. Set throttling to "Slow 3G"
4. Reload index.html
5. Verify in Network tab:
   - Hero image 1 loads immediately (high priority)
   - Other images show as "pending" until scrolled into view
   - Lazy images only load when needed
6. Scroll down the page slowly
7. Watch images load as they enter viewport
8. Verify no content shift or broken images
9. Check Lighthouse score improvement (note before/after)
</task>
</tasks>
