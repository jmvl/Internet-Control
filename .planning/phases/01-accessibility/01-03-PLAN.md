# 01-03: ARIA Labels & Landmarks

<objective>
Add semantic landmarks and ARIA labels to all pages, establishing proper document structure for screen reader users and improving accessibility.
</objective>

<context>
@websites/www.dinero.cash/index.html
@websites/www.dinero.cash/users.html
@websites/www.dinero.cash/agents.html
@websites/www.dinero.cash/assets/css/accessibility-improvements.css
@.planning/STATE.md
</context>

<tasks>
<task name="Add semantic landmarks to index.html">
1. Open `index.html`
2. Find hero/navigation section (line ~3, `<div class="hero">`)
3. Change to: `<header class="hero" role="banner">`
4. Find navigation element (`<nav role="navigation">` or add if missing)
5. Add: `aria-label="Main navigation"`
6. Find main content section (`<div class="page">`)
7. Change to: `<main class="page" id="main-content" role="main">`
8. Find footer section (`<div class="section footer">`)
9. Change to: `<footer class="section footer" role="contentinfo">`
10. Save index.html
</task>

<task name="Add ARIA labels to interactive elements">
1. Open `index.html`
2. Find menu button (has `class="menu-button"`)
3. Add: `aria-label="Toggle navigation menu" aria-expanded="false"`
4. Find feature boxes (`class="feature-box"`)
5. For each box, add appropriate `aria-label`:
   - `id="walletmodels"` → add `aria-label="View wallet models features"`
   - `id="payments-transfers"` → add `aria-label="View payments and transfers features"`
   - `id="merchant"` → add `aria-label="View merchant features"`
   - `id="onboarding"` → add `aria-label="View onboarding features"`
6. Find slider arrows and add `aria-label`:
   - Left arrow: `aria-label="Previous slide"`
   - Right arrow: `aria-label="Next slide"`
7. Find close buttons in modals and add `aria-label="Close dialog"`
8. Save index.html
9. Repeat for users.html and agents.html
</task>

<task name="Add aria-hidden to decorative elements">
1. Open each HTML file (index.html, users.html, agents.html)
2. Find background images without semantic meaning
3. Add `aria-hidden="true"` to:
   - Decorative image containers
   - Background hero images
   - Decorative icon SVGs that are purely visual
4. DO NOT add to informative images (screenshots, interface mockups)
5. Save all files
</task>

<task name="Add aria-required and invalid to form inputs">
1. Open `index.html` and find demo form
2. Find required inputs and add `aria-required="true"`
3. Add error message containers: `<div class="error-message" role="alert"></div>`
4. Add `aria-describedby` linking input to hint/error
5. Repeat for contact-us.html form
6. Save files
</task>
</tasks>
